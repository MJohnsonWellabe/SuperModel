# 🦎 Gecko MedSupp — Medicare Supplement Seriatim Projection Model

A fully browser-based, actuarial-grade Medicare Supplement projection model. No server required — runs entirely in the browser and is deployable to **GitHub Pages**.

![Gecko Orange Theme](public/favicon.svg)

## Features

- 📋 **7-page application**: Documentation, Database, Inputs, Assumptions, Calculations, Outputs, Aggregation
- 📊 **Seriatim projection engine**: Policy-by-policy monthly projection with full income statement
- 🔢 **Formula library**: Editable Python-style formulas for every IS/BS line
- 📁 **Excel upload**: Drag-and-drop `.xlsx` policy files (SheetJS)
- 🎛️ **Assumptions editor**: Scalar, by-duration, and keyed assumption tables
- 🔍 **Single-policy debugger**: Month-by-month view with formula traceability
- 📈 **Aggregation**: Income Statement + Balance Sheet + charts (Recharts)
- 💾 **Versioning**: Snapshot saves to browser storage (localStorage), export/import as zip
- 🟠 **Orange gecko theme**: Distinctive dark-orange brand aesthetic

## Model Logic (MVP)

```
For each policy, for each month t:

  inforce_eop[t] = inforce_bop[t] × (1 − lapse_rate[t])
  premium[t]     = inforce_bop[t] × premium_amount
  claims[t]      = inforce_bop[t] × claim_cost[t]
  commissions[t] = premium[t] × commission_rate[t]
  expenses[t]    = inforce_bop[t] × expense_per_policy + premium[t] × expense_pct_prem
  premium_tax[t] = premium[t] × prem_tax_rate
  nii[t]         = avg_reserve[t] × nii_rate
  reserve_change = reserve_eop − reserve_bop  (UPR proxy)

  pretax_income  = premium + nii − claims − commissions − expenses − premium_tax − reserve_change
  fit[t]         = max(0, pretax_income) × fit_rate
  net_income[t]  = pretax_income − fit
```

## Quick Start (Local Development)

### Prerequisites
- Node.js 18+
- npm 9+

### Setup

```bash
git clone https://github.com/YOUR_USERNAME/medsupp-model.git
cd medsupp-model

npm install
npm run dev
```

Open http://localhost:5173 in your browser.

### Load Sample Data

1. In the app, go to **Liability Inputs**
2. Click **"Load Sample Data (50 policies)"**
3. Navigate to **Assumptions** to review default rates
4. Go to **Calculations**, pick a policy, and view the debug projection
5. Go to **Outputs**, click **"Run All Policies"**
6. Go to **Aggregation** to view the Income Statement and charts

### Generate Excel Sample File

```bash
npm install  # if not already done
node generate-sample-data.js
# → sample-data/sample_policies.xlsx
```

Then drag-and-drop `sample-data/sample_policies.xlsx` into the Liability Inputs page.

## Deploying to GitHub Pages

### Method 1: Manual Build + Push

```bash
# Build into /docs folder
npm run build:docs

# Commit and push
git add docs/
git commit -m "Deploy to GitHub Pages"
git push origin main
```

Then in your GitHub repo:
1. Go to **Settings → Pages**
2. Source: **Deploy from a branch**
3. Branch: `main`, Folder: `/docs`
4. Save — your app will be live at `https://YOUR_USERNAME.github.io/medsupp-model/`

### Method 2: GitHub Actions (Automatic)

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm install
      - run: npm run build:docs
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./docs
```

## Scenario Versioning

### Save Snapshot
Click **"Save Snapshot"** in the top bar, enter a change note, and press Save.
- Snapshots are stored in browser localStorage (up to 20)
- View and restore from the Documentation page

### Export Scenario
Click **"Export"** in the top bar to download a `.zip` bundle containing:
- `scenario.json` — model metadata
- `inputs.csv` — policy data
- `assumptions.json` — all assumption tables
- `formulas.json` — formula library
- `outputs.csv` — run results (if available)

### Import Scenario
Click **"Import"** and select a previously exported `.zip` bundle to restore state.

## Excel File Format

Required columns:

| Column | Type | Required | Description |
|--------|------|----------|-------------|
| `policy_number` | string | ✅ | Unique policy identifier |
| `issue_year` | integer | ✅ | Policy issue year |
| `issue_month` | integer | ✅ | Issue month (1–12) |
| `premium_amount` | float | ✅ | Monthly premium ($) |
| `character_1..10` | string | ❌ | User-defined policy attributes |

## Actuarial Assumptions Reference

| Assumption | Default Type | Default Value |
|------------|-------------|---------------|
| Lapse Rate | By Duration | 0.5%/mo yr1, grading to 0.2%/mo yr3+ |
| Claim Cost | By Duration | $180/mo yr1 grading to $220+ |
| Commission Rate | By Duration | 20% yr1, 5% renewal |
| Expense Per Policy | Scalar | $12.50/mo |
| Expense % Premium | Scalar | 3.0% |
| Premium Tax Rate | Scalar | 2.0% |
| FIT Rate | Scalar | 21.0% |
| NII Rate | Scalar | 0.4%/mo |
